# Tower Breakers — 프로젝트 기술 명세서

> **작성일**: 2026-03-15
> **대상 버전**: main 브랜치 최신 워킹 카피
> **엔진**: Unity (URP)
> **언어**: C# (.NET Standard 2.1)

---

## 1. 프로젝트 개요

Tower Breakers는 **2D 횡스크롤 타워 돌파 액션 게임**이다. 플레이어는 타워의 각 층에서 밀려오는 적 대열을 공격, 패링, 스킬로 처리하며 상위 층을 향해 진행한다. 적에게 벽으로 밀리면 체력을 잃고, 체력이 0이 되면 게임 오버다.

### 1.1 핵심 게임 루프

```
[게임 시작]
    │
    ▼
[층 진입] ── 사전 스폰 (WAITING 상태) ── 입장 연출
    │
    ▼
[전투] ── 적 대열 전진 ── 플레이어 공격/패링/스킬
    │         │
    │         └── 벽 압착 → 데미지 → 사망 시 게임 오버
    │
    ▼
[층 클리어] ── 보상 (장비/체력) ── 다음 층 준비
    │
    └── 보스 층: 보스 등장 연출 → 페이즈 전투 → 클리어
```

## 2. 아키텍처 설계 원칙

### 2.1 핵심 원칙

## 3. 의존성 주입 (DI) 구조

### 3.1 VContainer 기반 모듈화

`GameLifetimeScope`가 최상위 등록기로서 9개의 DI 모듈을 구성한다.

```
GameLifetimeScope : LifetimeScope
    │
    ├── CoreDIModule
    │   ├── EventBus (Singleton)
    │   ├── GameStateMachine
    │   ├── CooldownSystem
    │   └── GameStates (Loading, Playing, GameOver)
    │
    ├── PlayerDIModule
    │   ├── PlayerModel, PlayerStateMachine
    │   ├── PlayerStates (Idle, Attack, Leap, Defend, Skill)
    │   ├── ISkillExecutor[] (Windstorm, Missile, Slash)
    │   └── PlayerView, PlayerPushReceiver
    │
    ├── EnemyDIModule
    │   ├── EnemyFactory (오브젝트 풀)
    │   ├── EnemySpawner
    │   └── ProjectileFactory
    │
    ├── CombatDIModule ── CombatSystem
    ├── TowerDIModule ── TowerManager
    ├── EnvironmentDIModule ── EnvironmentManager
    ├── UIDIModule ── HUDViewModel, EquipmentViewModel, GameOverViewModel
    ├── RewardDIModule ── RewardApplier
    └── SoundDIModule ── SoundManager, SoundPlayer
```

### 3.2 등록 방식

- **순수 C# 클래스**: `builder.Register<T>(Lifetime.Singleton)`
- **MonoBehaviour**: `builder.RegisterComponent(instance)`
- **ScriptableObject**: `builder.RegisterInstance(data)`
- **인터페이스 바인딩**: `builder.Register<T>().As<IT>()`

## 4. 이벤트 버스 시스템

### 4.1 설계

`EventBus`는 타입 기반 Pub/Sub 메시징 시스템으로, 시스템 간 직접 참조를 제거한다.

```csharp
public interface IEventBus
{
    void Publish<T>(T message) where T : struct;
    void Subscribe<T>(Action<T> action) where T : struct;
    void Unsubscribe<T>(Action<T> action) where T : struct;
}
```

**설계 결정 사항**:
- 이벤트는 `struct`로 강제하여 **힙 할당을 방지**
- `GetInvocationList()` 순회로 개별 핸들러 예외를 격리 (하나가 실패해도 나머지 실행)
- `Dictionary<Type, object>`로 타입별 델리게이트 체인 관리

### 4.2 이벤트 카탈로그

| 범주 | 이벤트 | 발행자 | 주요 구독자 |
|------|--------|--------|-----------|
| 게임 | `OnGameStart` | PlayingState | SoundManager |
| 게임 | `OnGameOver` | CombatSystem | GameStateMachine |
| 플레이어 | `OnPlayerDamaged` | CombatSystem | HUDViewModel, PlayerEffectView |
| 플레이어 | `OnPlayerPushed` | PlayerPushReceiver | CombatSystem |
| 플레이어 | `OnPlayerFloorChanged` | PlayingState | EnemyController, EnvironmentManager |
| 플레이어 | `OnPlayerAttackLanded` | PlayerAttackState | EnemyController |
| 플레이어 | `OnPlayerActionStarted` | PlayerLeapState 등 | EnemyController (동결 해제) |
| 플레이어 | `OnDefendActionTriggered` | PlayerDefendState | EnemyController (패링) |
| 적 | `OnEnemyDamaged` | EnemyDamageReceiver | EnemyController (보스 페이즈) |
| 적 | `OnEnemyKilled` | EnemyDamageReceiver | TowerManager, PlayerModel |
| 적 | `OnEnemyBuffRequested` | EnemyBuffState | EnemyController |
| 타워 | `OnFloorStarted` | PlayingState | EnemyController (대기→전진) |
| 타워 | `OnFloorCleared` | TowerManager | PlayingState |
| 연출 | `OnWallCrushOccurred` | CombatSystem | EnemyController (전원 동결) |
| 연출 | `OnHitEffectRequested` | CombatSystem | CombatEffectPresenter |
| 연출 | `OnDamageTextRequested` | CombatSystem | DamageTextSpawner |

## 5. 플레이어 시스템

### 5.1 PlayerModel (POCO)

플레이어의 순수 데이터 모델. Unity 의존성 없음.

**주요 데이터**:
- `m_currentLifeCount` / `m_maxLifeCount`: 횟수 기반 체력
- `m_currentWeapon`: 현재 무기 (`WeaponData`)
- `m_currentHelmet` / `m_currentBodyArmor`: 갑주 분리 시스템

**스탯 계산**: 기본 스탯 + 장비 보정을 합산하는 `Final*` 프로퍼티 제공.

```
최종 공격력 = PlayerData.BaseAttackPower + WeaponData.AttackPowerBonus
최종 이동속도 = PlayerData.BaseMoveSpeed + BodyArmor.MoveSpeedBonus
최종 밀림 저항 = BodyArmor.PushResistanceBonus
```

### 5.2 PlayerStateMachine (FSM)

| 상태 | 진입 조건 | 주요 동작 |
|------|----------|----------|
| `PlayerIdleState` | 기본 | 입력 대기, 액션 버튼 감시 |
| `PlayerAttackState` | 공격 버튼 | 무기별 공격 애니메이션, 히트박스 활성, 넉백 적용 |
| `PlayerLeapState` | 도약 버튼 | DOTween 백플립 연출, 다음 층 이동 |
| `PlayerDefendState` | 방어 버튼 | 패링 윈도우, 성공 시 적 넉백+기절 |
| `PlayerSkillState` | 스킬 버튼 | `ISkillExecutor` 전략 패턴으로 스킬 실행 |

### 5.3 스킬 시스템 (전략 패턴)

```csharp
public interface ISkillExecutor
{
    bool IsOnCooldown { get; }
    void Initialize(...);
    UniTask ExecuteAsync(CancellationToken ct);
}
```

| 구현체 | 설명 |
|--------|------|
| `WindstormSkillExecutor` | 전방 범위 공격 (다수 타격) |
| `MissileSkillExecutor` | 유도 미사일 발사 (`PlayerGuidedMissile`) |
| `SlashSkillExecutor` | 전방 관통 투사체 (`PlayerSlashProjectile`) |

스킬 사용 시 `EnemyPushLogic.EnableAllGroupCollidersForSkill()` 호출로 모든 적의 콜라이더를 일시 활성화하여 범위 타격 판정을 수행한다.

## 6. 적 시스템

### 6.1 적 타입 분류

```csharp
public enum EnemyType
{
    Normal,           // 일반 병사 — 기본 전진+밀기
    Tank,             // 탱커 — 높은 HP, 높은 밀기 힘
    SupportBuffer,    // 서포터(버프) — 후방에서 아군 힐
    SupportShooter,   // 서포터(사격) — 후방에서 투사체 발사
    Elite,            // 엘리트 — 강화된 일반 적
    Boss              // 보스 — BossFSM으로 별도 관리
}
```

### 6.2 EnemyController (코디네이터 패턴)

`EnemyController`는 적의 모든 하위 시스템을 조율하는 코디네이터이다.

**하위 시스템**:

| 시스템 | 역할 |
|--------|------|
| `EnemyStateMachine` | 상태 전이 관리 |
| `EnemyDamageReceiver` | 데미지 수신, HP 관리, 사망 처리 |
| `EnemyDebuffSystem` | 넉백, 슬로우, 기절 디버프 처리 |
| `EnemyPushLogic` | 군집 이동 및 플레이어 밀기 |
| `BossFSM` | 보스 전용 FSM (보스 타입만) |

**이벤트 구독 목록**:
- `OnPlayerFloorChanged`: 플레이어가 자신의 층에 도착하면 전진 시작
- `OnEnemyDamaged`: 보스인 경우 페이즈 전환 체크
- `OnPlayerAttackLanded`: 넉백 + 기절 적용
- `OnWallCrushOccurred`: 자기 층이면 전원 동결
- `OnPlayerActionStarted`: Leap 시 동결 해제
- `OnDefendActionTriggered`: 패링 넉백 + 기절
- `OnFloorStarted`: 대기 → 전진 전환
- `OnEnemyBuffRequested`: 힐 적용

### 6.3 EnemyStateMachine

```csharp
public interface IEnemyState
{
    void OnEnter();
    void OnExit();
    void OnTick();
}
```

| 상태 | 역할 |
|------|------|
| `EnemyWaitingState` | 사전 스폰 후 대기 (OnFloorStarted 대기) |
| `EnemyPushState` | 플레이어 방향 전진 + 밀기 |
| `EnemySupportPushState` | 서포터 전진 (후방 유지, 사격/버프 전환) |
| `EnemyShootState` | 투사체 발사 (SupportShooter) |
| `EnemyBuffState` | 아군 힐 버프 (SupportBuffer) |
| `EnemyStunnedState` | 기절 (일정 시간 후 복귀) |
| `EnemyFrozenState` | 동결 (벽 압착 시 전원 적용) |

**상태 팩토리**: `EnemyStateFactory`가 적 타입에 따라 필요한 상태만 생성한다. 보스는 빈 상태 생성 후 `BossFSM`이 관리를 인수한다.

### 6.4 오브젝트 풀링 (EnemyFactory)

```
UnityEngine.Pool.ObjectPool<EnemyView>
    │
    ├── createFunc: Instantiate + VContainer.Inject
    ├── actionOnGet: SetActive(true)
    ├── actionOnRelease: SetActive(false)
    └── actionOnDestroy: Object.Destroy
```

- 프리팹별(`EnemyData.EnemyName`) 독립 풀 관리
- `defaultCapacity: 5`, `maxSize: 30`
- 사망 시 `pool.Release(view)` 콜백으로 자동 반환

## 7. 군집 알고리즘 (핵심 시스템)

> 상세 내용은 `군집_알고리즘_기술명세서.md` 참조.

### 7.1 알고리즘 요약

**패턴**: 이중 연결 리스트 기반 기차 대열(Train Formation)

**핵심 구성 요소**:

| 구성 요소 | 클래스 | 역할 |
|-----------|--------|------|
| 대열 노드 | `EnemyPushLogic` | 앞/뒤 적 참조, 리더 탐색, 차단 판정 |
| 이동 실행기 | `EnemyMovementHelper` | 전진, 간격 유지, 플레이어 밀기 |
| 대열 구성기 | `EnemySpawner` | 스폰 시 연결 리스트 구축 |
| 플레이어 수신기 | `PlayerPushReceiver` | 밀림 힘 수신, 저항 계수 적용 |

### 7.2 알고리즘 원리

1. **연결 리스트 구축**: 스폰 순서대로 `SetAheadEnemy(previousEnemy)` 호출
2. **전파식 정지**: 리더가 막히면 `IsBlocked()` 재귀 호출로 후방까지 정지 전파
3. **간격 유지**: `pos.x = max(pos.x, ahead.x + TrainSpacing)` 하드 클램핑
4. **플레이어 밀기**: 접촉 시 위치 동기화 + 힘 기반 밀기 이중 전달
5. **사망 시 재연결**: `HandleDeath()`에서 앞뒤 노드 직접 연결

### 7.3 최적화 기법 요약

| 기법 | 복잡도 개선 | 구현 |
|------|-----------|------|
| 세대 번호 캐싱 | `GetLeader()` O(N) → O(1) | `s_generation` 전역 카운터 |
| 프레임 캐싱 | `IsBlocked()` O(N²) → O(N) | `m_lastBlockedFrame` |
| 콜라이더 선택 활성화 | 물리 연산 O(N) → O(1) | 리더만 활성화 |
| 정적 레지스트리 | 탐색 O(씬) → O(1) | `s_activeEnemies` |
| Transform 캐싱 | 프로퍼티 접근 최소화 | `m_cachedTransform` |
| Fisher-Yates Shuffle | 스폰 무작위화 O(N) | 클래스 필드 리스트 재사용 |

### 7.4 메모리 관리

Update 루프 내 **Zero GC Allocation** 달성:
- 모든 이동 연산이 스택/캐시 기반
- 재귀 호출 시에도 힙 할당 없음
- 정적 리스트 순회만 사용 (LINQ 미사용)

## 8. 보스 AI 시스템

### 8.1 BossFSM (상태 머신)

보스는 일반 적의 `EnemyStateMachine` 대신 전용 `BossFSM`으로 관리된다.

**상태 전이**:

```
BossIdleState ──(패턴 선택)──→ BossAttackState
       ↑                           │
       │                     (패턴 완료)
       └───────────────────────────┘

       ↓ (HP 임계값 도달)
BossPhaseChangeState ──(연출 완료)──→ BossIdleState (다음 페이즈)

       ↓ (패링/기절)
BossStunnedState ──(시간 경과)──→ BossIdleState

       ↓ (HP 0)
BossDeadState
```

### 8.2 페이즈 시스템

```csharp
public interface IBossPhase
{
    int PhaseIndex { get; }
    float HpThreshold { get; }       // 이 HP 비율 이하가 되면 다음 페이즈
    List<IBossPattern> Patterns { get; }  // 사용 가능한 패턴 목록
}
```

HP가 임계값 이하로 떨어지면 `CheckPhaseTransition()`이 호출되어 다음 페이즈로 전환된다.

### 8.3 패턴 실행 (전략 패턴)

```csharp
public interface IBossPattern
{
    string PatternName { get; }
    UniTask ExecuteAsync(EnemyController controller, CancellationToken ct);
}
```

**보스별 패턴 목록**:

| 보스 | 패턴 | 설명 |
|------|------|------|
| 고블린 수장 | JumpPattern | 도약 후 착지 충격파 |
| | SwingPattern | 근접 휘두르기 |
| | BombPattern | 폭탄 투척 |
| | LightningPattern | 번개 소환 |
| | BuffPattern | 아군 강화 |
| | TotemPattern | 토템 설치 |
| 크라켄 | ArtilleryPattern | 포격 (낙하물) |
| | TentaclePattern | 촉수 공격 |
| | SummonTentaclePattern | 촉수 소환 |
| 로봇 | SwordDashPattern | 검 돌진 |
| | ShieldActivePattern | 방패 활성화 |
| | GunnerAttackPattern | 총격 |

### 8.4 BossSkillSelector (행동 트리)

현재 페이즈의 패턴 목록에서 적절한 패턴을 선택한다. 행동 트리(BT) 기반으로 조건 노드와 실행 노드를 조합하여 상황에 맞는 패턴을 결정한다.

## 9. 전투 시스템

### 9.1 CombatSystem (POCO)

```csharp
public class CombatSystem : IInitializable, IDisposable
```

VContainer의 `IInitializable`을 구현하여 DI 완료 후 자동 초기화된다.

**핵심 흐름: 벽 압착 데미지**

```
OnPlayerPushed (이벤트 수신)
    │
    ├── PlayerModel.TakeDamage(1)        — 고정 1 데미지
    ├── Publish(OnPlayerDamaged)          — HUD 갱신
    ├── Publish(OnDamageTextRequested)    — 데미지 텍스트
    ├── Publish(OnWallCrushOccurred)      — 적 전원 동결
    ├── Publish(OnHitEffectRequested)     — 카메라 쉐이크 + 히트스톱
    └── PlayerModel.IsDead?
        └── Publish(OnGameOver)           — 게임 오버
```

### 9.2 CombatEffectPresenter

타격 연출 담당 (MonoBehaviour):
- 카메라 쉐이크 (DOTween)
- 히트스톱 (Time.timeScale 제어)
- 역경직 효과

## 10. 타워 진행 시스템

### 10.1 TowerManager

```
TowerData (SO)
    └── List<FloorData>
            ├── EnemyPrefabData: EnemyData
            ├── EnemyCount: int
            ├── SpawnInterval: float
            ├── TrainSpacing: float
            ├── SpawnPackets: List<SpawnPacket>
            └── IsBossFloor: bool
```

**층 진행 흐름** (`PlayingState`):

```
1. PrepareNextFloor(currentFloor)
   └── EnemySpawner.SpawnFloorEnemiesAsync(isPreSpawn: true)
       └── 적 생성 → WAITING 상태 배치

2. PlayPlayerEnterAsync()
   └── 입장 애니메이션

3. IsBossFloor? → PlayBossIntroAsync()
   └── BossIntroCutscene 연출

4. Publish(OnFloorStarted)
   └── 모든 WAITING 적 → PUSH 상태 전환

5. PrepareNextFloor(currentFloor + 1)
   └── 다음 층 사전 스폰

6. 적 전멸 → OnFloorCleared
   └── 보상 → 다음 층 진입
```

## 15. 성능 최적화 총정리

| 범주 | 기법 | 효과 |
|------|------|------|
| **메모리** | Update 루프 Zero Allocation | GC 스파이크 제거 |
| **메모리** | 오브젝트 풀링 (적, 투사체, 데미지 텍스트) | Instantiate/Destroy 최소화 |
| **물리** | 콜라이더 선택적 활성화 | 물리 연산 O(N) → O(1) |
| **탐색** | 정적 레지스트리 (s_activeEnemies) | FindObjectsOfType 제거 |
| **캐싱** | Transform 참조 캐싱 | 프로퍼티 접근 오버헤드 제거 |
| **캐싱** | 세대 번호 기반 리더 캐싱 | GetLeader() O(N) → O(1) |
| **캐싱** | 프레임 단위 차단 판정 캐싱 | IsBlocked() O(N²) → O(N) |
| **비동기** | UniTask (코루틴 대체) | GC.Alloc 최소화 |
| **트위닝** | DOTween (자체 풀링) | 애니메이션 코드 간소화 |
| **업데이트** | 층별 선택적 Update | 비활성 층 적 연산 차단 |
